🔴 المشكلة #1: طباعة التوكن في الـ Console (خطر أمان)
   📍 الموقع: main.py السطور 151، 203، 204
   🔍 الوصف: البرنامج يطبع التوكن في الـ console وهذا يعتبر ثغرة أمان خطيرة جداً
   ⚠️  المخاطر:
      • يمكن لأي شخص يرى الـ console أن يسرق التوكن
      • التوكن قد يظهر في السجلات (logs) على الخادم
      • قد يتم تسجيل التوكن في Docker logs أو الـ CI/CD logs
   
   ✅ الحل:
      • إزالة أو إخفاء التوكن من الـ print statements
      • طباعة فقط أول 10 أحرف من التوكن بـ [REDACTED] للبقية
      • استخدام logging module بدلاً من print للعمليات الحساسة
      • إضافة متغير بيئي HIDE_SENSITIVE_INFO لإخفاء البيانات الحساسة
   
   📝 الخطوات:
      1. فتح main.py
      2. استبدال جميع print statements للتوكن بـ print(f"🔑 التوكن: {token[:10]}...")
      3. إضافة logging module للعمليات المهمة
      4. اختبار أن البوت يشتغل بدون طباعة التوكن الكامل

🔴 المشكلة #2: عدم وجود ملف .env
   📍 الموقع: جذر المشروع
   🔍 الوصف: المتغيرات البيئية الحساسة (التوكن) لا يتم حفظها في ملف .env
   ⚠️  المخاطر:
      • التوكن يتم تخزينه في config.json وهذا غير آمن
      • قد يتم رفع التوكن عن طريق الخطأ على GitHub
      • صعوبة في إدارة التوكن المختلفة لبيئات مختلفة
   
   ✅ الحل:
      • إنشاء ملف .env يحتوي على:
        TOKEN=your_token_here
        PREFIX=!
        DEBUG=False
      • تحديث main.py ليقرأ من .env أولاً ثم config.json
      • إضافة .env إلى .gitignore
      • استخدام python-dotenv library لقراءة المتغيرات
   
   📝 الخطوات:
      1. تثبيت python-dotenv: pip install python-dotenv
      2. إنشاء ملف .env في جذر المشروع
      3. تحديث main.py ليستخدم dotenv.load_dotenv()
      4. اختبار أن البوت يقرأ من .env
